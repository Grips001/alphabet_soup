  <action>
    Create BeltSystem (pure TS, no Phaser):

    1. Internal storage: Map<string, Belt> keyed by "x,y"
    2. processingOrder: Belt[] — sorted array rebuilt on add/remove

    3. addBelt(belt: Belt): void — stores belt, rebuilds processing order
    4. removeBelt(x, y): Belt | null — removes belt, rebuilds processing order
    5. getBeltAt(x, y): Belt | null

    6. tick(): void — the core simulation step:
       a. Call snapshotPosition() on every item currently on a belt
       b. Process belts in reverse propagation order (farthest downstream first)
       c. For each belt with an item: check next belt in direction. If next exists and next.item is null, move item to next belt (set next.item = item, current.item = null, update item.tile). If blocked or no next, item stays.

    7. getNextBelt(belt: Belt): Belt | null — returns the belt at the tile in belt's output direction

    8. rebuildProcessingOrder(): void — For each belt, compute its "distance from end" (how many belts downstream until dead end). Sort belts by distance ascending (farthest downstream = shortest distance = processed first). For Phase 2, linear chains only — no splits/merges. Use a simple traversal: for each belt, walk downstream counting steps.

    9. placeBeltPath(path: Array<{x: number, y: number}>, buildingSystem: BuildingSystem): Belt[] — Convert a drag path into placed belts:
       a. For each consecutive pair of tiles, determine direction from current to next
       b. Create Belt with that direction
       c. For corner tiles (direction changes), compute variant using computeBeltVariant(inputDir, outputDir)
       d. Place via buildingSystem.place()
       e. Update variants of neighboring existing belts that now connect
       f. Return array of created belts

    CRITICAL: Process farthest-downstream first to prevent double-advance bug. Test this explicitly with a 3-belt chain and 2 items.

  <action>
    Create QuarrySystem (pure TS, no Phaser):

    1. Constructor takes BuildingSystem and BeltSystem references
    2. quarries: Quarry[] — all active quarries

    3. initializeQuarries(buildingSystem: BuildingSystem): void
       - For each entry in QUARRY_DEFINITIONS, create a Quarry instance
       - Place via buildingSystem.place()
       - Store in quarries array

    4. tick(): void — called once per TickEngine tick:
       a. For each quarry: call quarry.tickProduction() (increment timer)
       b. Check if quarry.canProduce():
          - Look up belt at quarry.outputTile via beltSystem.getBeltAt()
          - If no belt: quarry is backpressured (nowhere to output), skip
          - If belt exists but belt.item is not null: quarry is backpressured, skip
          - If belt exists and belt.item is null: create new LetterItem(quarry.letter, outputTile), set belt.item = newItem, call quarry.resetTimer(), clear backpressured flag
       c. Update quarry.backpressured flag based on output state

    5. getQuarries(): readonly Quarry[] — for renderer access
  </action>
