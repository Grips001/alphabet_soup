<task type="auto">
  <name>Task 2: ItemRenderer with alpha interpolation</name>
  <files>
    src/game/renderers/ItemRenderer.ts
  </files>
  <action>
    **ItemRenderer.ts** (Phaser):
    1. Constructor takes scene and BeltSystem
    2. Maintains Map<LetterItem, Phaser.GameObjects.Container> — each container has a colored rectangle + text

    3. syncItems(): void — called each tick (or each frame)
       a. Get all items from belts (iterate all belts in beltSystem, collect those with non-null item)
       b. For new items not in the map: create Container with:
          - Rectangle (ITEM_SIZE x ITEM_SIZE) filled with LETTER_COLORS[item.letter]
          - Text centered on rectangle showing the letter character (dark color, bold, ~10px)
          - Depth: DEPTH_ITEMS
       c. For items no longer on any belt: destroy their container, remove from map

    4. update(alpha: number): void — called every frame from GameScene.update()
       a. For each tracked item:
          - Compute from = grid.tileToCenter(item.previousTile.x, item.previousTile.y)
          - Compute to = grid.tileToCenter(item.tile.x, item.tile.y)
          - Set container position: x = from.x + (to.x - from.x) * alpha, y = from.y + (to.y - from.y) * alpha
       b. This produces smooth sliding between tile positions

    5. removeItem(item: LetterItem): void — destroy container, remove from map
    6. destroy(): clean up all containers

    CRITICAL: Use item.previousTile (snapshotted at tick time) as "from" and item.tile (current) as "to". This is what makes items slide smoothly instead of popping.
  </action>

    <automated>bun run build</automated>
  </verify>
  <done>ItemRenderer draws colored letter blocks that smoothly interpolate between tile positions using alpha. Compiles without errors.</done>
