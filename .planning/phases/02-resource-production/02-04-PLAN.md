  <name>Task 2: ToolbarUI, GhostRenderer, and GameScene input wiring</name>

  <action>
    **ToolbarUI.ts** (Phaser, UI camera):
    1. Constructor takes scene, PlacementSystem, and the UI camera reference
    2. Create a bottom toolbar bar:
       a. Background rectangle spanning viewport width, ~48px tall, at bottom of screen
       b. Dark semi-transparent background (e.g., rgba(30,30,40,0.85))
    3. Tool buttons (3 total):
       a. Belt tool (hotkey: 1) — icon: rectangle with arrow/chevron
       b. Demolish tool (hotkey: 2 or Delete key) — icon: X mark
       c. Each button: ~40x40 rectangle with icon, setInteractive()
       d. on('pointerover'): lighten background
       e. on('pointerdown'): call placementSystem.selectTool(toolType)
    4. Active tool highlighting: brighter border/background on selected tool button
    5. update(): sync visual highlight to placementSystem.currentTool
    6. All toolbar objects added to UI camera, ignored by main camera
    7. Handle resize: reposition toolbar on scene resize event

    **Keyboard hotkeys** (in GameScene):
    - Key 1: select Belt tool
    - Key 2 or Delete: select Demolish tool
    - ESC: deselect (ToolType.None)
    - Use scene.input.keyboard.on('keydown-*') — call preventDefault() to avoid browser defaults

    **GhostRenderer.ts** (Phaser):
    1. Constructor takes scene and Grid reference
    2. Create a semi-transparent rectangle (belt-sized: TILE_SIZE x TILE_SIZE) — the ghost sprite
    3. Set alpha to 0.5, depth to DEPTH_GHOST
    4. update(placementSystem, buildingSystem): void — called every frame:
       a. If currentTool is None: hide ghost
       b. If currentTool is Belt:
          - Show ghost at grid.tileToCenter(cursorTile)
          - Tint green (0x00ff00) if isValidPlacement, red (0xff0000) if not
          - If invalid, show an X mark overlay (small graphics cross)
       c. If currentTool is Demolish:
          - Show ghost with red tint at cursor tile
          - Different visual: maybe a red X instead of building preview
    5. Ignored by UI camera, visible on main camera (it's in world space)

    **GameScene.ts** wiring:
    1. Create PlacementSystem in create()
    2. Create ToolbarUI and GhostRenderer in create()
    3. Register pointer events:
       a. scene.input.on('pointermove'): get world coords via cameras.main.getWorldPoint(pointer.x, pointer.y), call placementSystem.updateCursor(worldX, worldY, grid)
       b. scene.input.on('pointerdown'): if left click and tool selected, call placementSystem.startDrag()
       c. scene.input.on('pointermove') during drag: call placementSystem.updateDrag()
       d. scene.input.on('pointerup'): call placementSystem.endDrag(), then:
          - If tool was Belt: call beltSystem.placeBeltPath(path, buildingSystem), then for each new belt call beltRenderer.addBelt(belt)
          - If tool was Demolish: for each tile in path, call buildingSystem.demolish(tx, ty), beltRenderer.removeBelt(tx, ty), clean up items via itemRenderer
          - After any place/demolish: update adjacent belt variants and their renderers
    4. In update():
       a. Call ghostRenderer.update(placementSystem, buildingSystem)
       b. Call toolbarUI.update()
    5. Register keyboard hotkeys (1, 2, Delete, ESC) with preventDefault()
    6. Remove the temporary test belt from Plan 03 (if present)

    **Demolish behavior per CONTEXT.md decisions:**
    - Click to demolish single building
    - Click+drag to demolish multiple buildings along path
    - Items on demolished belts vanish (call itemRenderer.removeItem for each belt's item before removing belt)
    - Quarries cannot be demolished (they are fixed map features) — OR allow demolish but items stop producing. Per CONTEXT.md "click to demolish single building" without quarry exception, allow quarry demolish for now.
    - After demolish, update neighbor belt variants (they may revert from corner to straight)

    **Important UX details from CONTEXT.md:**
    - Single click (pointerdown + pointerup on same tile) places one belt or demolishes one building
    - Click-drag paints a belt path or demolishes along the drag path
    - Placement on occupied tiles is BLOCKED (ghost shows red, placement skipped)
    - Belt direction follows drag direction (from previous tile to current tile)
