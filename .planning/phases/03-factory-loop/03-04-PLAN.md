  <files>

  <behavior>
    New ToolTypes:
    - ToolType.Inserter, ToolType.Splitter, ToolType.Underground, ToolType.Assembler added to enum

    Rotation:
    - placementDirection defaults to North
    - rotateDirection() cycles N -> E -> S -> W -> N
    - Selecting a tool that doesn't use rotation (Belt, Demolish) does not reset direction

    Multi-tile validity:
    - isValidPlacement accepts optional footprint override: getFootprint(tool, tx, ty, direction) returns array of tiles
    - Belt: [{x: tx, y: ty}] (unchanged)
    - Inserter: [{x: tx, y: ty}] (1x1)
    - Assembler: 9 tiles from (tx, ty) to (tx+2, ty+2)
    - Splitter: 2 tiles based on direction (perpendicular to flow)
    - Underground entry/exit: [{x: tx, y: ty}] (1x1)
    - isValidFootprint(footprint, buildingSystem) checks ALL tiles are in bounds and unoccupied

    Two-click underground placement:
    - pendingUndergroundEntry: { x, y, direction } | null
    - When tool=Underground and pendingEntry is null: first click sets pendingEntry
    - When tool=Underground and pendingEntry exists: second click places exit (if valid range + same direction)
    - getUndergroundExitRange(entry, direction): returns valid tile positions for exit (1 to UNDERGROUND_MAX_DISTANCE tiles in direction)
    - isValidUndergroundExit(exitTile): checks range, bounds, not occupied
    - If exit invalid, do not place — keep pendingEntry
    - Switching away from Underground tool clears pendingEntry
    - ESC clears pendingEntry

    Click-to-configure:
    - getConfigTarget(tx, ty, buildingSystem): returns building at tile if tool=None, else null
    - This is a read-only method — GameScene decides what to do with the returned building

    Single-click placement (not drag) for Inserter, Assembler, Splitter:
    - These tools use click placement, not drag. startDrag/endDrag still works but path will be length 1.
    - Belt and Demolish continue to use drag placement.
  </behavior>
  <action>
    Extend the existing PlacementSystem. Add new ToolType enum values. Add:
    - `placementDirection: InserterDirection` (default North) — imported from Inserter entity
    - `rotateDirection(): void` — cycles direction
    - `pendingUndergroundEntry: { x: number; y: number; direction: InserterDirection } | null`
    - `getFootprint(tool: ToolType, tx: number, ty: number, direction: InserterDirection): Array<{x: number, y: number}>`
    - `isValidFootprint(footprint: Array<{x: number, y: number}>, buildingSystem: BuildingSystem): boolean`
    - `getConfigTarget(tx: number, ty: number, buildingSystem: BuildingSystem): Building | null` — only when currentTool === None

    Override `selectTool()` to clear pendingUndergroundEntry when switching away from Underground tool.

    Keep existing Belt/Demolish drag behavior unchanged. New tools (Inserter, Assembler, Splitter) work with single-click (drag path length 1).

    Write tests for new behavior first (RED), then implement (GREEN). Keep existing tests passing.
