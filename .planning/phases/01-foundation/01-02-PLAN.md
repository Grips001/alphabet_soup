  <name>Task 2: Implement CameraController with zoom-toward-cursor</name>
  <files>src/game/systems/CameraController.ts, src/game/systems/CameraController.test.ts</files>
  <action>
Create `src/game/systems/CameraController.ts`:

1. Constructor takes `scene: Phaser.Scene`

2. **WASD + Arrow key panning** using SmoothedKeyControl:
   - Create cursor keys AND WASD keys
   - Since SmoothedKeyControl accepts one key per direction, create a wrapper: for each direction, create a composite key object where `isDown` returns true if EITHER the arrow or WASD key is pressed
   - Config: acceleration=CAMERA_ACCELERATION, drag=CAMERA_DRAG, maxSpeed=CAMERA_MAX_SPEED from constants
   - Call `controls.update(delta)` in the update method

3. **Scroll wheel zoom toward cursor** (per research Pattern 2):
   - Listen to `scene.input.on('wheel', ...)`
   - Get world point under cursor before zoom change
   - Apply zoom delta (0.1 per scroll step), clamp to ZOOM_MIN..ZOOM_MAX
   - Use lerped zoom: store target zoom, lerp toward it each frame for smoothness
   - After zoom, adjust scrollX/scrollY to keep the world point under cursor stable
   - Handle `camera.preRender()` or equivalent for matrix recalculation (verify method exists in Phaser 3.90; fallback: use `camera.dirty = true`)

4. **Middle-click drag**:
   - On pointerdown (button === 1, middle mouse): record starting pointer position and camera scroll
   - On pointermove while middle button held: calculate delta from start, apply to camera scroll
   - On pointerup: release
   - Prevent browser auto-scroll: add DOM-level `addEventListener('mousedown', ...)` on canvas with `event.preventDefault()` for button 1 (per research Pitfall #4)

5. **Camera bounds**: Set `camera.setBounds(0, 0, WORLD_SIZE, WORLD_SIZE)` to prevent seeing void (per user decision)

Create `src/game/systems/CameraController.test.ts`:
   - Test zoom clamping logic (ensure zoom stays within ZOOM_MIN..ZOOM_MAX)
   - Test that zoom delta calculation is correct (positive deltaY = zoom out, negative = zoom in)
   - Note: Most camera behavior requires Phaser runtime; test the pure math helpers only
