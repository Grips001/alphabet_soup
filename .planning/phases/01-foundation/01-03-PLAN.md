  <action>
1. Create `.github/workflows/ci.yml` (per research code example):
   ```yaml
   name: CI
   on:
     push:
       branches: [main]
     pull_request:
       branches: [main]
   jobs:
     check:
       runs-on: ubuntu-latest
       steps:
         - uses: actions/checkout@v4
         - uses: oven-sh/setup-bun@v2
         - run: bun install --frozen-lockfile
         - run: bun run type-check
         - run: bun run test
   ```

2. Create `vercel.json` (minimal config for Vite SPA):
   ```json
   {
     "buildCommand": "bun run build",
     "outputDirectory": "dist",
     "framework": "vite"
   }
   ```

3. Deploy to Vercel using CLI:
   - Run `vercel link` to connect the project (if not already linked)
   - Run `vercel --yes` for initial deployment
   - Verify the deployment URL works
   - Set up Vercel GitHub integration for auto-deploy on push (if not already configured via Vercel dashboard, use `vercel git connect`)

4. Push to GitHub to trigger CI:

   - If CI fails, fix and re-push
