1. Performance Is a Gameplay Feature
Players will always build something bigger than you expect.
So the engine must support:
Massive scale
Deterministic simulation
Consistent tick timing
Key principles to follow:
60 UPS deterministic simulation
Highly optimized entity updates
Chunk-based world simulation
Multithreaded pathfinding
Efficient memory layouts

2. Systems Must Be Legible at Scale
A player should be able to look at the factory and understand what is happening.
The game should provide:
Clear item visuals
Belt direction readability
Inserter animation feedback
Production graphs
Why this matters:
In complex systems games:
Confusion destroys creativity.
If a player cannot diagnose problems visually, the game becomes frustrating instead of satisfying.
The game should make debugging intuitive:
Items backing up on belts
Assemblers idle
You can see the bottleneck immediately.

3. Every System Should Compose With Every Other System
Everything interacts with everything else.
Examples:
Belts feed assemblers
Inserters move anything
Nothing exists in isolation.
Bad factory games create parallel mechanics that don’t integrate.
Example problems:
Separate transport systems that don’t interact
Crafting systems disconnected from automation
Logistics layers that bypass factory logic
Instead we should create a unified simulation language.

4. Logistics Is the Real Game
In factory games:
Production is trivial. Logistics is the puzzle.
Our game should deeply embrace this.
Players solve problems like:
Throughput limits
Belt balancing
The interesting problems emerge from moving items, not crafting them.
Games that overfocus on crafting recipes miss the point.

5. The Player Must Be Able to Rebuild the World
The game should let players:
Deconstruct anything
Blueprint anything
Copy entire factories
Upgrade in place
Late game tools should include:
Blueprint books
Upgrade planners
Without this:
Large factories become painful to evolve.
Many newer factory games underestimate how critical this is.

6. Debugging Tools Are Essential
Advanced players need introspection.
The game should include:
Production statistics
Consumption graphs
These tools transform the player from:
Builder → Engineer

7. The Best Factory Games Are “Thinking Games”
What should make our game special:
It’s not about production.
It’s about mental models.
Players constantly reason about:
Flow systems
Queues
Throughput
Resource graphs
Feedback loops
It’s essentially systems engineering disguised as a game.
